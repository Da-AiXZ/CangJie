# 阶段2：数据模型修正 + 提示词广场重写

**目标**：修复与后端字段不一致的数据模型，消除运行时解码失败风险。本阶段不新增功能，只修字段+接线。

## 2.1 CircuitBreaker 字段重写

| 维度 | 内容 |
|------|------|
| 原版实现 | `api/autopilot.ts:30-40` AutopilotCircuitBreakerData 接口 |
| iOS现状 | `Models/AutopilotModels.swift` CircuitBreakerStatus 字段名全错 |
| 严重度 | 高（解码失败或字段nil，熔断器状态展示错误） |

**原版字段（必须对齐）**：
- `status: string`（iOS错为 `state`）
- `error_count: number`（iOS错为 `failure_count`）
- `max_errors: number`（iOS错为 `threshold`）
- `last_error: { message: string, timestamp: string, context?: string }`（iOS只有 `last_failure_at` 时间戳，缺 message/context）
- `error_history?: AutopilotErrorRecord[]`（iOS缺失）

**要补**：
1. 改字段名：state→status, failure_count→error_count, threshold→max_errors
2. last_error 改为嵌套结构（message+timestamp+context）
3. 新增 error_history 数组（AutopilotErrorRecord 模型）
4. 新增 AutopilotErrorRecord 模型（message/timestamp/context）

**核验**：grep iOS代码确认字段名与原版一致；解码后端返回的JSON不报错。

## 2.2 BibleGenerationStatus/Feedback 字段重写

| 维度 | 内容 |
|------|------|
| 原版实现 | `api/bible.ts` getBibleStatus / getBibleGenerationFeedback |
| iOS现状 | `Models/BibleModels.swift` 字段完全不同，对错了接口 |
| 严重度 | 高（运行时解码失败） |

**原版字段（必须对齐）**：
- getBibleStatus 返回 `{ exists: boolean, ready: boolean, novel_id: string }`
- getBibleGenerationFeedback 返回 `{ novel_id: string, error: string|null, stage: string, at: string }`

**iOS现状（全错）**：
- BibleGenerationStatus 是 `{ status, stage?, progress?, message? }` — 字段完全不同
- BibleGenerationFeedback 是 `{ feedback?, suggestions? }` — 字段完全不同

**要补**：重写两个模型，字段对齐原版。

**核验**：调后端 getBibleStatus 接口，iOS 能正确解码 exists/ready/novel_id。
